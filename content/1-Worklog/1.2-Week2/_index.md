### Week 2 Objectives:

* Understand AWS networking architecture and Virtual Private Cloud (VPC).
* Learn how to configure secure and scalable network infrastructure on AWS.
* Practice implementing networking components and load balancing services.
* Gain hands-on experience with Lab 03.

### Week 2 Achievements:

* Successfully built a basic AWS networking environment consisting of:

  * One Virtual Private Cloud (VPC)
  * Public and Private Subnets distributed across multiple Availability Zones

* Configured network security components, including:

  * Security Groups for EC2 instances
  * Network ACLs for subnet-level traffic control

* Successfully deployed and configured:

  * Internet Gateway
  * NAT Gateway
  * Route Tables

* Understood how traffic routing works within an AWS VPC and how networking components interact.

* Learned the principles of:

  * VPC Peering
  * Transit Gateway
  * AWS VPN
  * AWS Direct Connect
  * VPC Endpoint

* Gained practical experience with **Elastic Load Balancer**, including:

  * Application Load Balancer (ALB)
  * Network Load Balancer (NLB)
  * Gateway Load Balancer (GWLB)

* Understood how load balancing improves application availability, scalability, and fault tolerance.

### Plan for Next Week:

* Launch EC2 instances inside the VPC created during Week 2.

* Configure a NAT Gateway to allow private instances secure outbound Internet access.

* Deploy an EC2 Instance Connect Endpoint for secure instance access.

* Perform connectivity testing and validate the complete Lab 04 networking architecture.
